# スケジュール更新ガイド

このガイドでは、ウェブサイトのスケジュールを更新する方法を説明します。
プログラミングの知識は不要です！

## 📋 更新方法

### 1. ファイルを開く

`schedule.json` というファイルをテキストエディタで開きます。

推奨エディタ:
- **Windows**: メモ帳、Visual Studio Code
- **Mac**: テキストエディット、Visual Studio Code

### 2. スケジュールを編集する

ファイルを開くと、以下のような形式で書かれています:

```json
{
  "events": [
    {
      "title": "第123回 松戸ボードゲーム会",
      "date": "2025年12月7日（日）",
      "time": "13:00 - 18:00",
      "venue": "松戸市民会館 301会議室",
      "capacity": "12名",
      "fee": "500円",
      "status": "募集中",
      "twiplaUrl": "https://twipla.jp/events/123456",
      "description": "年末に向けて楽しいゲーム会を開催します！初心者大歓迎です。"
    }
  ]
}
```

### 3. 各項目の説明

| 項目 | 説明 | 例 |
|------|------|------|
| `title` | イベントのタイトル | "第123回 松戸ボードゲーム会" |
| `date` | 開催日 | "2025年12月7日（日）" |
| `time` | 開催時間 | "13:00 - 18:00" |
| `venue` | 開催場所 | "松戸市民会館 301会議室" |
| `capacity` | 定員 | "12名" |
| `fee` | 参加費 | "500円" |
| `status` | 募集状況 | "募集中", "募集予定", "満席", "開催済み" |
| `twiplaUrl` | TwiPlaのURL | "https://twipla.jp/events/123456" |
| `description` | イベントの説明 | "年末に向けて..." |

### 4. 新しいイベントを追加する

複数のイベントを追加する場合は、カンマ(`,`)で区切って追加します:

```json
{
  "events": [
    {
      "title": "第123回 松戸ボードゲーム会",
      "date": "2025年12月7日（日）",
      ...
    },
    {
      "title": "第124回 松戸ボードゲーム会",
      "date": "2025年12月21日（日）",
      ...
    }
  ]
}
```

### 5. イベントを削除する

削除したいイベントの `{ }` ブロック全体を削除します。
複数のイベントがある場合は、カンマに注意してください。

### 6. ファイルを保存する

編集が完了したら、ファイルを保存します。

## ⚠️ 重要な注意点

### ❌ これをしないでください

1. **ダブルクォーテーション(`"`)を消さない**
   - 悪い例: `title: 第123回` ← `"`がない
   - 良い例: `"title": "第123回"`

2. **カンマ(`,`)を忘れない**
   - 複数のイベントや項目の間には必ずカンマが必要です
   - 最後のイベントの後にはカンマは不要です

3. **波括弧(`{ }`)や角括弧(`[ ]`)を消さない**
   - これらは構造を保つために必要です

### ✅ 正しい例

```json
{
  "events": [
    {
      "title": "第123回 松戸ボードゲーム会",
      "date": "2025年12月7日（日）",
      "time": "13:00 - 18:00",
      "venue": "松戸市民会館 301会議室",
      "capacity": "12名",
      "fee": "500円",
      "status": "募集中",
      "twiplaUrl": "https://twipla.jp/events/123456",
      "description": "年末に向けて楽しいゲーム会を開催します！"
    }
  ]
}
```

## 🎨 ステータスの色分け

`status` フィールドに以下の値を設定すると、自動的に色が変わります:

- **"募集中"** → 🟢 緑色（募集中）
- **"募集予定"** → 🔵 青色（近日募集開始）
- **"満席"** → 🔴 赤色（満員）
- **"開催済み"** → ⚫ グレー（終了）

## 🔗 TwiPlaリンクについて

- `twiplaUrl` にURLを入れると、「詳細を見る」ボタンが表示されます
- URLがない場合（空欄 `""`）は、「近日公開予定」と表示されます

例:
```json
"twiplaUrl": "https://twipla.jp/events/123456"  ← リンクあり
"twiplaUrl": ""  ← リンクなし（近日公開）
```

## 🆘 困ったときは

### エラーが出た場合

1. **ダブルクォーテーション(`"`)が全て対になっているか確認**
2. **カンマ(`,`)が正しく配置されているか確認**
3. **波括弧(`{ }`)や角括弧(`[ ]`)が正しく閉じられているか確認**

### JSONの文法チェック

オンラインツールで文法をチェックできます:
- [JSONLint](https://jsonlint.com/) - JSONの文法チェックツール

ファイルの内容をコピーして貼り付け、「Validate JSON」をクリックすると、エラーがあれば教えてくれます。

```json
  "date": "20XX年XX月XX日（○）",

  "fee": "XXX円",
  "status": "募集中",

  "description": "イベントの説明文をここに書きます。"

```

## 🎯 よくある質問

**Q: スケジュールを更新したのに、サイトに反映されません**
A: ブラウザのキャッシュをクリアして、ページを再読み込みしてください（Ctrl+F5 または Cmd+Shift+R）

**Q: 古いイベントは削除すべきですか？**
A: はい、過去のイベントは定期的に削除することをおすすめします。サイトが見やすくなります。

**Q: イベントの順番は重要ですか？**
A: 上から順番に表示されるので、新しいイベントを上に配置することをおすすめします。
